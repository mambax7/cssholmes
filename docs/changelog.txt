<h5>3.0.0 Alpha 2 [NOT RELEASED]</h5>
Dev: XOOPS 2.5.12, PHP 8.4, PHP 8.5.0
<hr>
- added permission checks to all admin workbench pages and actions (mamba)
- added a new "cssHolmes Admin" permission for granular control over who can access the admin workbench (mamba)
- added **Diagnose (third row):**
        - `overflow` — highlight every element with overflowing content
        - `z-index` — show z-index values as colored labels on all positioned elements
        - `grid` — show CSS Grid and Flexbox containers with column/row lines
        - `proto` / `deuter` / `trita` / `achro` — simulate color blindness types
- added CSS Grid and Flexbox diagnostics to the front-end inspection overlay (mamba)


<h5>3.0.0 Alpha 1 [2026-03-26]</h5>
Dev: XOOPS 2.5.12, PHP 8.4, PHP 8.5.0
<hr>
- split the work into a new cssHolmes module identity with its own templates, preload, and admin workbench (mamba)
- added profile-based overlay modes for html5, xtf-theme, xtf-widget, a11y, and layout diagnostics (mamba)
- added a modern floating toolbar with inspect, measure, copy text/json, export selected inspection, undo, widget jump, and slot jump (mamba)
- added token-aware XTF theme inspection and local token preview editing when a theme.json manifest is available (mamba)
- added typography editing for font size, line height, letter spacing, weight, and alignment preview (mamba)
- added XTF/XMF-oriented admin workbench flows for theme scan, widget HTML scan, import review, patch drafts, preview/apply, and rollback (mamba)
- added server-side analyzers for theme manifest, design tokens, templates, accessibility, and widget output (mamba)
- added a new front-end hub page plus a modern XTF/XMF testsuite alongside the legacy HTML5 testsuite (mamba)
- added explicit generic XOOPS fallback messaging when no XTF theme.json manifest is available, while keeping generic inspection and diagnostics usable (mamba)


<h5>2.0.0 Beta 1  [ NOT RELEASED]</h5>
